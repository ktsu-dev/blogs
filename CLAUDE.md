## Project Overview

This is a **markdown-based technical blog** for ktsu.dev. Blog posts are written as markdown files with YAML frontmatter, and a PowerShell script auto-generates the `README.md` as a blog index.

## Key Rules

- **Never edit `README.md` directly** — it is auto-generated by `scripts/Rebuild-BlogIndex.ps1` and will be overwritten.
- Blog posts **must** be placed in `content/blog/` as `.md` files.
- Only posts with `status: published` in their frontmatter appear in the index.

## Commands

```bash
# Rebuild the blog index (regenerates README.md from post frontmatter)
pwsh scripts/Rebuild-BlogIndex.ps1

# Or on Windows without pwsh in PATH
scripts\rebuild-blog.bat
```

## Blog Post Format

Every post must start with YAML frontmatter between `---` delimiters:

```yaml
---
title: "Post Title"
author: "Matt Edmondson"
created: 2025-07-18
modified: 2025-07-18
status: published          # draft | review | published
description: "One-line summary used in the index"
categories: ["Development", "C#"]
tags: ["csharp", "topic-tag"]
keywords: ["SEO keyword phrases"]
slug: "url-friendly-post-name"
---
```

Required fields: `title`, `author`, `created`, `status`, `description`, `categories`, `tags`, `slug`.

## Architecture

```
content/blog/          # All blog posts (markdown with YAML frontmatter)
scripts/
  Rebuild-BlogIndex.ps1  # Parses frontmatter from all posts, regenerates README.md
  rebuild-blog.bat       # Windows convenience wrapper
.github/workflows/
  refresh-blog-index.yml # Runs Rebuild-BlogIndex.ps1 on push to main, commits updated README.md
README.md              # AUTO-GENERATED blog index — do not edit
```

### Index Rebuild Pipeline

1. Push to `main` triggers `.github/workflows/refresh-blog-index.yml`
2. Workflow runs `Rebuild-BlogIndex.ps1` which scans `content/blog/*.md`
3. Script parses YAML frontmatter, filters to `status: published` posts, sorts by `created` date descending
4. Generates README.md with: latest posts listing, posts by category, posts by tag groups, blog stats
5. If README.md changed, commits with `[skip ci]` to avoid recursive triggers

### Tag Grouping

The index script maps individual tags into display groups (defined in the `$tagGroups` hashtable in `Rebuild-BlogIndex.ps1`). When adding posts with new tags, update `$tagGroups` if the tags should appear in the "Posts by Tags" section.
